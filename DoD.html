DoD (Definition of Done):

  •  All Acceptance Criteria met – Mandatory
    •  contains at least 1 Test Case per 1 Acceptance Criteria – Recommended
  • All task are "Closed" – Mandatory
    •  Meets Continuous Delivery Requirements – Mandatory
  • Code Review is done – Mandatory if code changes
  •  No show-stoppers (no critical bugs) – Mandatory, the recommendation is no bugs
  • Deployed and test completed in QA – Mandatory
  • Accepted by the PO – Mandatory
